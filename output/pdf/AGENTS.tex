\section{AGENTS.md - Kaleido FlowTwin engineering
contract}\label{agentsmd---kaleido-flowtwin-engineering-contract}

This file is normative for any human or agent implementing the MVP.

\subsection{1. Mission}\label{1-mission}

Build a read-only predictive operations layer for Kaleido that starts
from sparse operational event data, produces immediate process
intelligence, and promotes a sequential/world-model component only when
it adds measured value.

Primary task:

\begin{quote}
Predict remaining time and material deviation risk for one Trace Port
operation/shift, with calibrated uncertainty, useful lead time and an
auditable explanation.
\end{quote}

The result must complement Trace Port, Shipping Board and Freight
Intelligence. Do not build a competing generic logistics platform.

\subsection{3. Claim policy}\label{3-claim-policy}

Never claim \texttt{SOTA}, production-ready, causal, ROI, savings,
accuracy, early warning or successful deployment without generated
evidence.

Allowed states:

\begin{itemize}
\tightlist
\item
  \texttt{planned}: specified but not implemented;
\item
  \texttt{smoke\_only}: pipeline test, not business evidence;
\item
  \texttt{diagnostic}: internal development result, test may be reused;
\item
  \texttt{claim\_eligible}: frozen protocol, clean commit, held-out
  test, provenance;
\item
  \texttt{pilot\_shadow}: running read-only with operator review;
\item
  \texttt{validated\_pilot}: pre-agreed acceptance gates passed.
\end{itemize}

Every result statement must name:

\begin{itemize}
\tightlist
\item
  dataset/export version and hash;
\item
  split protocol;
\item
  model and baselines;
\item
  metric and uncertainty;
\item
  number of seeds;
\item
  threshold selection method;
\item
  whether test influenced a choice;
\item
  claim state.
\end{itemize}

Public/synthetic datasets prove pipeline competence only. Kaleido value
requires Kaleido data.

\subsection{4. Non-negotiable data
rules}\label{4-non-negotiable-data-rules}

\subsubsection{4.1 Actions are not
context}\label{41-actions-are-not-context}

An \texttt{action} is a timestamped decision that an operator/controller
could change. Cargo type, vessel, customer, weather and shift identity
are context. Static metadata must not support causal or counterfactual
language.

Every adapter implements:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{build\_manifest()}
\NormalTok{validate\_timestamps()}
\NormalTok{classify\_fields()}
\NormalTok{build\_object\_graph()}
\NormalTok{build\_grouped\_splits()}
\NormalTok{run\_leakage\_audit()}
\end{Highlighting}
\end{Shaded}

Training fails closed when the audit fails. An explicit
\texttt{-\/-unsafe-debug} mode may create watermarked smoke artifacts
that can never enter reports.

\subsubsection{4.2 Plan revisions are immutable
events}\label{42-plan-revisions-are-immutable-events}

Never replace planned dates with latest values. Store each revision and
its \texttt{valid\_from}. A prediction at time \texttt{t} may only see
the plan revision available at \texttt{t}.

\subsubsection{4.3 Keep object identity}\label{43-keep-object-identity}

Project, operation, shift, cargo unit, resource, vessel, document and
incident are separate object types. Do not flatten them into duplicated
case rows without a reversible mapping.

\subsubsection{4.4 Censoring and unknown
outcomes}\label{44-censoring-and-unknown-outcomes}

Incomplete operations are censored, not failures and not exact remaining
time. Unknown incident status remains unknown. Use survival methods when
appropriate.

\subsubsection{4.5 No random row split}\label{45-no-random-row-split}

Default priority:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  chronological future;
\item
  held-out project/client/port;
\item
  held-out cargo/operation type;
\item
  grouped cross-validation.
\end{enumerate}

All prefixes/events from one operation remain in one partition.

\subsection{5. Baselines before world
models}\label{5-baselines-before-world-models}

Required floors:

\begin{itemize}
\tightlist
\item
  global and group median;
\item
  persistence/rule thresholds;
\item
  Cox/Kaplan-Meier or another censoring-aware model;
\item
  regularized linear/logistic model;
\item
  quantile gradient boosting/CatBoost;
\item
  GRU/TCN;
\item
  ProcessTransformer;
\item
  object-centric graph baseline when relationships are available.
\end{itemize}

Do not remove a baseline because it wins.

Event-JEPA is permitted after M0/M1/M2 data and baseline gates. Required
ablations:

\begin{itemize}
\tightlist
\item
  current prefix only;
\item
  context only;
\item
  correct actions;
\item
  shuffled actions;
\item
  action only;
\item
  no JEPA objective;
\item
  one horizon vs multi-horizon;
\item
  no anticolapse regularizer;
\item
  frozen vs fine-tuned encoder;
\item
  object graph vs flattened trace.
\end{itemize}

If correct actions do not beat shuffled actions, do not claim action
value.

\subsection{6. Architecture boundaries}\label{6-architecture-boundaries}

\subsubsection{Ingestion}\label{ingestion}

Read-only connectors, export parsing, source lineage and checksums. No
model features beyond canonical normalization.

\subsubsection{Semantic/event layer}\label{semanticevent-layer}

Canonical IDs, timestamps, units, field roles, OCEL-like relationships
and versioned plans. Unknown semantics stay unknown.

\subsubsection{Analytics}\label{analytics}

Process discovery, conformance, variants, bottlenecks and descriptive
reports. Must operate without ML training.

\subsubsection{Prediction}\label{prediction}

Baselines, sequential models, Event-JEPA and uncertainty. Models never
write source data.

\subsubsection{Simulation/scenarios}\label{simulationscenarios}

Discrete-event simulation is separate from learned action ranking. A
simulated saving is not a realized saving. Learned scenarios are
restricted to observed support and approved actions.

\subsubsection{Serving/UI}\label{servingui}

Versioned read-only API, dashboard, export and audit trail. Every
prediction names its source event cutoff and plan revision.

\subsection{10. Metrics}\label{10-metrics}

Do not optimize only AUROC or next-event accuracy.

Remaining time:

\begin{itemize}
\tightlist
\item
  MAE, median AE, pinball loss;
\item
  P50/P90 coverage and interval width;
\item
  per operation/cargo/project and worst group.
\end{itemize}

Risk:

\begin{itemize}
\tightlist
\item
  AUPRC, event precision/recall;
\item
  false alerts per shift/100 operations;
\item
  lead time and missed events;
\item
  Brier/ECE and reliability.
\end{itemize}

Process:

\begin{itemize}
\tightlist
\item
  conformance/deviation;
\item
  waiting time and rework;
\item
  variant coverage;
\item
  bottleneck stability.
\end{itemize}

Business:

\begin{itemize}
\tightlist
\item
  time saved in reporting;
\item
  avoidable wait/overtime under shadow simulation;
\item
  operator acceptance/usefulness;
\item
  estimated vs realized value clearly separated.
\end{itemize}

Thresholds are selected on validation only.

\subsection{11. Adaptation}\label{11-adaptation}

Never update the only reference model. Maintain:

\begin{itemize}
\tightlist
\item
  frozen reference;
\item
  adaptive shadow/adapters;
\item
  trust region;
\item
  uncertainty/update gate;
\item
  replay/anti-forgetting;
\item
  signed/versioned artifact;
\item
  rollback;
\item
  explicit reset after process redesign.
\end{itemize}

Log every accepted/rejected update.

\subsection{12. Security}\label{12-security}

\begin{itemize}
\tightlist
\item
  minimum-privilege read-only credentials;
\item
  secrets from approved secret store only;
\item

\end{itemize}

\subsection{15. Definition of done}\label{15-definition-of-done}

The MVP is done when:

\begin{itemize}
\tightlist
\item
  a fresh environment installs with one documented command;
\item
  a Trace Port-like export is audited without manual code edits;
\item
  process maps and data-quality report regenerate;
\item
  baselines run on frozen grouped splits;
\item
  prediction intervals and operational metrics are reported;
\item
  Event-JEPA is either validated incrementally or honestly rejected;
\item
  shadow replay exposes prediction cutoff, interval and reasons;
\item
  no source write capability exists;
\item
  security/data agreement checklist is complete;
\item
  an operator review is documented;
\item
  all tables are generated from artifacts;
\item
  limitations and \texttt{what\ this\ does\ not\ prove} are explicit.
\end{itemize}

At the end of each task report: hypothesis, changes, tests, evidence,
limitations and next falsifiable step.
