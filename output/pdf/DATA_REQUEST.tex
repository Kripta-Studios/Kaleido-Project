\section{Peticion minima de datos para la
reunion}\label{peticion-minima-de-datos-para-la-reunion}

\subsection{Objetivo}\label{objetivo}

Determinar en menos de una semana si existe un caso predictivo viable,
sin solicitar un volcado masivo ni acceso productivo.

\subsection{Paquete minimo}\label{paquete-minimo}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Diccionario de campos/export de Trace Port.
\item
  Tres a cinco operaciones cerradas anonimizadas:

  \begin{itemize}
  \tightlist
  \item
    una nominal;
  \item
    una lenta/desviada;
  \item
    una con incidencia si existe.
  \end{itemize}
\item
  Plan original y revisiones disponibles.
\item
  Eventos/timestamps y relaciones con operacion, turno, carga y recurso.
\item
  Resultado final: fin, desviacion e incidencia.
\item
  Captura o export de dashboard solo si no expone informacion sensible.
\item
  Descripcion de que decision se toma cuando aparece riesgo.
\end{enumerate}

Formatos aceptables: CSV, XLSX, JSON, Parquet o export de API. No se
necesita base de datos ni credenciales para el scan.

\subsection{Seudonimizacion}\label{seudonimizacion}

Kaleido puede sustituir:

\begin{itemize}
\tightlist
\item
  cliente, proyecto, buque, persona y recurso por IDs estables;
\item
  notas por categorias o texto redactado;
\item
  fotos por metadata o excluirlas inicialmente;
\item
  importes por rangos.
\end{itemize}

No debe eliminar timestamps, relaciones ni revisiones, porque son la
senal del proceso.

\subsection{Preguntas por tabla/export}\label{preguntas-por-tablaexport}

\begin{itemize}
\tightlist
\item
  ¿Que representa una fila?
\item
  ¿Cual es la clave primaria?
\item
  ¿Que timezone usa?
\item
  ¿El timestamp es de ocurrencia o registro?
\item
  ¿Puede haber duplicados/reintentos?
\item
  ¿Se actualiza una fila o se crea historial?
\item
  ¿Que campos existian en el momento del evento?
\item
  ¿Que outcome se conoce solo al final?
\item
  ¿Que IDs enlazan tablas?
\item
  ¿Que valores son decisiones y quienes pueden cambiarlos?
\end{itemize}

\subsection{Conteos que bastan antes del
volcado}\label{conteos-que-bastan-antes-del-volcado}

\begin{itemize}
\tightlist
\item
  numero de proyectos/operaciones/turnos por mes;
\item
  fecha minima/maxima;
\item
  eventos por operacion;
\item
  porcentaje con plan y fin real;
\item
  porcentaje con incidencias;
\item
  tipos de carga/operacion;
\item
  numero de puertos/clientes;
\item
  frecuencia de replanificacion;
\item
  cobertura de recursos/equipos.
\end{itemize}

\subsection{Respuesta del Data Evidence
Scan}\label{respuesta-del-data-evidence-scan}

EVOCON devolvera:

\begin{itemize}
\tightlist
\item
  mapa de datos;
\item
  score de readiness por objetivo;
\item
  issues y acciones de instrumentacion;
\item
  caso candidato y baseline;
\item
  estimacion de esfuerzo/computo;
\item
  \texttt{GO\_PREDICTIVE}, \texttt{INSTRUMENT\_FIRST} o
  \texttt{NO\_FIT};
\item
  propuesta cerrada del piloto, sin obligacion de continuar.
\end{itemize}

\subsection{Lo que no se solicita
inicialmente}\label{lo-que-no-se-solicita-inicialmente}

\begin{itemize}
\tightlist
\item
  acceso de escritura;
\item
  credenciales productivas;
\item
  datos personales sin necesidad;
\item
  fotos originales;
\item
  todos los productos a la vez;
\item
  anos de datos sin saber la estructura;
\item
  aprobacion para controlar operaciones.
\end{itemize}
